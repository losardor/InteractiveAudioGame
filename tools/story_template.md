# STORY TITLE

## Meta
**Setting:** Where does the story take place? Describe it as a soundscape.
**Listener role:** Who is the listener and why are they here?
**Central question:** The theme the story orbits — not the plot, but the deeper question.
**Echo:** Optional. A small act early in the story that returns transformed at the end.

<!-- OPTIONAL FIELDS REFERENCE
The fields below can be added to any Option or Ending block where needed.
You do not have to use all of them. Simple stories may need none of them.

In Options:
  **Flag set:** flag_name         — this option silently sets a named flag when chosen
  **Flag required:** flag_name    — this option is only available if the named flag was previously set
  **Gates:** Ending A, Ending B   — only these endings are reachable after this option is chosen

In Endings:
  **Reached by:** description     — describe which combination of choices leads here (for your reference)

Flag names are freeform. Use anything that is meaningful to you: echo, trust, sacrifice, etc.
The parser treats flag names as case-insensitive strings.

Routing strategy:
  Use Gates on an upstream option to restrict which endings are reachable from that branch.
  Use Flag required on a later option only when a specific earlier action (not a branch) gates it.
  Prefer Gates over Flag required when the restriction follows naturally from a binary choice —
  this keeps the logic in one place and avoids redundant flag checks downstream.
